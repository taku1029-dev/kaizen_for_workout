# ADR-003: Data Migration Strategy — New Start

- **ステータス**: Accepted
- **日付**: 2026-06-23
- **決定者**: taku1029

## コンテキスト

現在 Google Spreadsheet に筋トレ記録が蓄積されている。新アプリへのデータ移行戦略として、CSV インポート・Google Sheets API 双方向同期・新規スタートの 3 案を検討した。

## 決定

**新規スタート**（アプリは今日から記録を開始し、既存 Spreadsheet は参照用に保持する）。

## 理由

- Spreadsheet のカラム構造（種目名が列ヘッダー、セット内容がセル内テキスト）は非正規化されており、SwiftData モデルへの変換にパース処理が必要
- 過去データの主な用途は「振り返り参照」であり、分析機能のベースラインとして必須ではない
- 移行コスト（CSV パーサー + マッピングロジック + バリデーション）に対して得られる価値が小さい
- アプリのローンチを早めることの方が長期的価値が高い

## 却下した代替案

| 案 | 却下理由 |
|----|---------|
| CSV 一括インポート | Spreadsheet のフォーマット（セル内の "60kg×10" 表記など）のパース実装コストが高い |
| Google Sheets API 双方向同期 | OAuth 認証・リアルタイム同期の複雑性。Google API への恒久的依存が生じる |

## 結果

- **ポジティブ**: 実装スコープが明確に絞られ、コア機能（記録・分析）の品質向上に集中できる
- **ネガティブ**: 過去の記録はアプリ上で参照できない
- **緩和策**: 既存 Google Spreadsheet はアーカイブとして保持し削除しない
- **将来の拡張**: 要望が高まれば CSV インポート機能を後から追加できる設計にしておく（`Services/ImportService.swift` のプレースホルダーを残す）
